1. Platform Overview

Lexington Betty Smokehouse is a catering ordering platform built for Chef Dominique Leach's BBQ catering business in Chicago. The platform allows customers to build custom catering orders, select from pre-built packages, and submit orders or request quotes — all through a guided, mobile-friendly ordering wizard.

Tagline: "Best BBQ in Chicago" — as seen on Good Morning America

Key Features

  • Guided Ordering Wizard — 6-step flow from event type to checkout
  • Build Your Own or Packages — Flexible ordering modes
  • Smart Portions — Auto-calculates quantities based on headcount
  • Equipment & Extras — Sternos, racks, utensils with smart suggestions
  • Take Us Home Upsells — Retail add-on products at checkout
  • Admin Dashboard — Full menu management, order tracking, analytics
  • Automated Emails — Order/quote confirmations via Resend
  • QuickBooks Integration — Auto-invoice creation with payment links
  • Kitchen Printouts — Packing slips and full order PDFs
  • Betty VIP — Customer loyalty program (Coming Soon)

Data Fallback Strategy

The application uses a fallback chain for resilience:

Supabase DB → Hardcoded Data → In-Memory Cache

If the database is unavailable, the app falls back to hardcoded products in lib/products.ts and lib/packages.ts, ensuring the ordering flow never breaks.

4. Customer Order Flow

The ordering process follows a guided wizard with 6 steps, plus checkout.

1. Event Type → 2. Event Info → 3. Guests & Budget → 4. Order Type → 5. Build Order → 6. Equipment → Checkout

Step 1: Event Type

Customer selects their event category:

  • BBQ Catering (Lunch) — Full smokehouse menu with meats, sides, desserts
  • Desserts & Drinks — Sweet treats and beverages only

Note: Breakfast has been removed from the current menu.

Step 2: Event Info

Collects event details: name, date, time, venue address, and special instructions. These carry forward to checkout and are pre-filled if already entered.

Step 3: Guests & Budget

Customer sets their headcount and optionally selects a budget range per person. The budget tracker in the cart shows real-time whether they're on track, under, or over budget.

Step 4: Order Type

Two options:

  • Build Your Own — Select individual items from the full menu
  • Packages — Choose a pre-built catering package with fixed per-person pricing

Step 5: Build Order

The main menu selection screen. Products are organized by category sections (Meats, Sliders, Sides, Desserts, Drinks). The cart sidebar on desktop (or drawer on mobile) shows running totals, per-person cost, and budget tracking. Quantities auto-adjust based on headcount.

Step 6: Equipment & Extras

Smart suggestions for serving equipment based on the order:

  • Pan count suggests sternos and catering racks
  • Headcount suggests utensil sets
  • Items: Sterno ($2), Catering Rack ($12), Serving Utensils ($2), Utensil Sets ($0.50)

Checkout

Two-step checkout process:

  1. Details — Contact info, delivery address, event details, delivery type (delivery or free local pickup), buffet setup option, special instructions
  2. Review & Submit — Full order review with option to "Request a Quote" or "Place Order"
Validation: A prominent red message box appears at the top of the form listing all missing required fields, with auto-scroll to the message.

After Submission

Save to DB → Send Email → Create QB Invoice → Confirmation Page

12. QuickBooks Integration

Overview

When QuickBooks Online is connected, the platform automatically creates invoices for new orders and provides customers with online payment links.

Connection Flow

Admin clicks Connect → OAuth2 Authorization → Intuit Login → Tokens Stored in DB

Order-to-Invoice Flow

  1. Customer places order (not a quote)
  2. System checks if QB is connected
  3. Finds or creates QB customer by email
  4. Creates invoice with all line items + delivery fee
  5. Enables online credit card and ACH payment
  6. Retrieves payment link from QB
  7. Updates order record with invoice ID and payment link
  8. Sends invoice email via QB's email system
Quotes skip QB: Quote requests do not create QuickBooks invoices. Only confirmed orders trigger the invoicing flow.

Webhook Handling

QuickBooks sends webhook notifications when invoices are updated or voided:

  • Invoice Update — Verifies invoice balance = $0, then marks order as "paid"
  • Invoice Void — Marks the linked order as "cancelled"
  • Signature verification using HMAC-SHA256 with timing-safe comparison
  • Always returns HTTP 200 to prevent retry storms

Security Measures

  • CSRF-safe OAuth state (random token in httpOnly cookie)
  • Token revocation on disconnect (calls Intuit revocation endpoint)
  • Automatic token refresh (5-minute buffer before expiry)
  • API versioning (minorversion=65) on all calls
  • Input sanitization on customer email queries
  • Admin-only access to connect/disconnect endpoints

Admin Page

Located at /admin/quickbooks. Shows connection status, company ID, token expiration, and connect/disconnect buttons.

15. Getting Started (Developer Guide)

Prerequisites

  • Node.js 18+ and npm
  • Git
  • Access to the GitHub repository

Local Setup

# Clone the repository
git clone https://github.com/rdawsonsdp/lexbetty.git
cd lexbetty

# Install dependencies
npm install

# Copy environment variables
cp .env.example .env.local
# Edit .env.local with your actual keys

# Start the development server
npm run dev
# App runs on http://localhost:3001

Key Directories

/app                    # Next.js pages and API routes
  /admin/               # Admin dashboard pages
  /api/                 # Server-side API endpoints
  /checkout/            # Checkout page
  /order-confirmation/  # Post-order confirmation

/components             # React components
  /admin/               # Admin panel components
  /catering/            # Ordering wizard components
  /layout/              # Header, Footer, Layout
  /ui/                  # Shared UI components (Card, Button, etc.)

/context                # React Context providers
  CateringContext.tsx    # Main ordering state management

/lib                    # Utilities and business logic
  /email/               # Resend email integration

  /supabase/            # Database client and helpers
  products.ts           # Product catalog (fallback data)
  packages.ts           # Package definitions
  pricing.ts            # Pricing calculation engine
  types.ts              # TypeScript type definitions
  order-pdf.ts          # PDF generation utilities

/public/images          # Logo assets and food photography

Common Commands

npm run dev       # Start development server (port 3001)
npm run build     # Production build
npm run start     # Start production server
npm run lint      # Run ESLint

Deployment

Push to main branch → Vercel auto-deploys to lexingtonbettycatering.com.

git add .
git commit -m "Your changes"
git push origin main
# Vercel will automatically build and deploy
